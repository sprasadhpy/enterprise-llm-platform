Current Architecture

Enterprise Authentication · Multi-Model Routing · Retrieval Orchestration · AI Governance · Production Observability

Platform Overview

PoC architecture: Entain employees access the platform via a web browser. The frontend handles authentication (Cognito SSO), load balancing, and serves React UI + Streamlit Dashboard. Microservices handle chat completions, conversations, messages, files, and feedback. LiteLLM routes to Claude and Nova models. RAG pipeline uses Bedrock Knowledge Base and OpenSearch Serverless. Data persists in DynamoDB and S3.

Production Architecture (VPC)

Production VPC architecture: Full deployment within AWS VPC. EKS orchestrates containers. ALB handles traffic distribution. ReactUI frontend connects to all microservices (Chat Completions, Files, Conversations, Messages, Dashboard, Feedback). LiteLLM gateway routes to Claude Family, Nova Family (via Bedrock) and GPT Family (via Azure OpenAI endpoint). RAG powered by OpenSearch Serverless Index + Bedrock Knowledge Base. S3 buckets for guardrails, file uploads, and feedback storage.

How We Launched EntainGPT

From security concern to enterprise-wide AI assistant in 12 weeks

Released: 13 June 2025

EntainGPT began as Prometheus, an initiative to replace unregulated ChatGPT usage across 25,000 employees. The core problem: staff were using personal ChatGPT accounts, exposing PII, HR data, and confidential information to external systems with no governance or audit trail.

We built a secure, enterprise-grade AI assistant platform as a viable alternative, with proper authentication, multi-model routing, monitoring, and compliance baked in from day one.

Phase 1: Basic Interactive UI

  • ChatGPT-like stateless UI for text interactions
  • SSO authentication with user profiles
  • Basic usage monitoring and audit logging
  • Model selection and routing (Claude, GPT, Gemini)
  • Session management (current session only)
  • Response latency target: <3 seconds
  • SSO authentication time: <1.5 seconds

Phase 2: Session Management & Context

  • Session affinity and conversation history
  • Expanded context window capabilities
  • Enhanced monitoring and audit trail
  • Document storage with Entain data integration

Phase 3: Advanced Features

  • Fine-grained model selection and optimization
  • Advanced security features and PII detection
  • Department-specific customizations
  • Integration with internal systems and documents

September 2025

📓Week 1: Project Kickoff

1 Sept 2025

Define the vision and scope of the enterprise AI platform.

Questions

  • What problem are we solving?
  • Who are the primary users?
  • What does "enterprise-ready" actually mean?
  • What are the non-functional requirements?

Decisions

  • AI assistant rather than a generic chatbot.
  • Security-first architecture.
  • Modular design to support multiple models in the future.

Deliverables

  • Initial architecture diagram
  • Technology evaluation
  • Project roadmap

Building enterprise AI starts with understanding business workflows, not choosing a language model.

📓Week 2: First Working Prototype

8 Sept 2025

Build the first end-to-end conversational prototype.

Built

  • Basic chat interface
  • Initial LLM integration
  • Streaming responses

Challenges

  • Generic responses
  • No enterprise knowledge
  • No conversation history

The next milestone should focus on grounding responses using enterprise knowledge.

Improve factual accuracy.
